System architecture

End-to-end flow for local PDF question answering.

1. Ingestion

PDFs are read with PyMuPDF, normalized to plain text, then split into overlapping character windows (chunks). Each chunk becomes one row in the vector store.

2. Embeddings & index

Sentence Transformers embed every chunk. Vectors are stored in a FAISS IndexFlatL2 for exact nearest-neighbour search at query time.

3. Retrieval

Your question is embedded with the same model. The top‑k closest chunks (configurable in Workspace) are concatenated into a single context block.

4. Generation

Context and question are sent to Ollama ({{ llm_model }}). The model must follow the policy in the prompt: answer only from context or decline.
