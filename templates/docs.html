How DocQuery Works

A simple, fully local Retrieval-Augmented Generation (RAG) system

1. Document Ingestion

When you upload a PDF:

  • PyMuPDF extracts text from all pages
  • Text is cleaned and split into smart overlapping chunks
  • Each chunk preserves context for better answers

2. Embeddings & Vector Store

Each chunk is converted into a numerical vector using {{ embed_model }}.

These vectors are stored in FAISS (local vector database) for fast similarity search.

3. Retrieval (Semantic Search)

When you ask a question:

  • Your question is converted into a vector
  • FAISS finds the most relevant chunks from your document
  • You can control how many chunks are retrieved in Settings

4. Answer Generation

The retrieved chunks + your question are sent to the selected Ollama model ({{ llm_model }}).

The model generates an answer based only on your document (with source awareness).

Complete Data Flow

PDF Upload
    ↓
Text Extraction (PyMuPDF)
    ↓
Chunking (Overlapping)
    ↓
Embedding ({{ embed_model }})
    ↓
FAISS Vector Index
    ↓
User Question → Embedding → Similarity Search → Top-k Chunks
    ↓
Context + Question → Ollama LLM
    ↓
Final Answer + Sources

Key Benefits

✅ 100% Local & Private
✅ No internet required after setup
✅ Your data never leaves your PC
✅ Switch models anytime
✅ Persistent memory (documents saved)
